## Overview

Perfin is a personal finance management app — a monorepo with a SvelteKit frontend and a Convex serverless backend.

## Commands

**Root (runs all packages in parallel):**
```bash
bun run dev    # Start all services
bun run build  # Build all packages
```

**Frontend only (`packages/web`):**
```bash
bun run dev      # Vite dev server
bun run check    # Type-check with svelte-check
bun run lint     # Prettier + ESLint
bun run format   # Auto-format with Prettier
```

**Backend only (`packages/backend`):**
```bash
bun run dev   # convex dev (local Convex server)
```

## Architecture

### Packages
- `packages/web` — SvelteKit frontend
- `packages/backend` — Convex backend (database + server functions)
- `packages/shared` — Shared TypeScript constants/utilities

### Frontend (`packages/web/src`)
- **Routing**: File-based SvelteKit routing. Protected app pages live under `routes/(app)/`. The root `/` is the auth page (login/signup).
- **Auth guard**: `hooks.server.ts` extracts a JWT token from cookies into `event.locals.token`. `requireLogin()` / `requireUnauthenticated()` enforce access control in `+layout.server.ts` / `+page.server.ts`.
- **State**: No global store. Uses Svelte 5 runes (`$state`, `$derived`) for component state, Convex's `useQuery()` for real-time server data, and Svelte context (`src/context.ts`) for sharing account data across the app layout.
- **Data mutations**: Use `useConvexClient()` and call `client.mutation(api.some.fn, args)` imperatively.

### Backend (`packages/backend/convex`)
- **Schema**: `schema.ts` defines `accounts` and `transactions` tables.
- **Functions**: `accounts.ts` and `transactions.ts` export Convex queries and mutations. Auth-related functions are in `auth.ts` / `auth.config.ts`.
- **Generated types**: `_generated/api.ts` is auto-generated by `convex dev` — never edit manually.
- **Importing backend API on frontend**: Use `@perfin/backend/convex/_generated/api`.

### Authentication
- Uses **better-auth** with a Convex database adapter and Google OAuth + email/password providers.
- Backend: `convex/auth.ts` sets up the auth instance. Required env vars: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `SITE_URL`.
- Frontend client: `src/lib/authClient.ts` — `createAuthClient()` with the Convex plugin.
- HTTP handler: `routes/api/auth/[...all]/+server.ts` handles all `/api/auth/*` routes.

### Styling
- Tailwind CSS v4 (configured via `@tailwindcss/vite` plugin, not a config file).
- UI primitives from `bits-ui`; icons from `lucide-svelte`.
